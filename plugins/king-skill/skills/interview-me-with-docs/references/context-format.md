# CONTEXT.md 格式(统一语言术语表)

`CONTEXT.md` 是某个上下文的领域词典——**只放术语,不放实现**。

## 结构
```md
# {上下文名}

{一两句描述}

## Language

**术语**:
定义(最多一两句,说它「是什么」而非「做什么」)。
_Avoid_: 要舍弃的近义词
```

## 规则
- **有主见**:近义词里选最合适的一个作规范术语,其余列到 `_Avoid_`。
- **定义要紧**:一两句说清「是什么」,别写「做什么」。
- **只收上下文专有术语**:通用编程概念不收,除非在本领域有特殊含义。自问:这个概念是本上下文专有,还是放之四海皆准?
- **逻辑分组**:术语多时用子标题聚类;同一主题的少量术语用扁平列表即可。
- **纯术语表**:零实现细节,不当 spec、不当草稿本。

## 单上下文 vs 多上下文
- **单上下文**:根目录放一个 `CONTEXT.md`。
- **多上下文**:根目录建 `CONTEXT-MAP.md`,列出各上下文位置 + 它们如何关联(含 Contexts 与 Relationships:数据流、共享类型)。
- **判定**:先找 `CONTEXT-MAP.md`,再找根 `CONTEXT.md`,都没有则懒创建。
